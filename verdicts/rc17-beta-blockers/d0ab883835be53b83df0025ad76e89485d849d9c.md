## Findings

### CC-RC17-006 — Major: release-posture guard is bypassed by ordinary Markdown line wrapping

**Dimensions:** Tests, Documentation

**Evidence:** `evaluate_release_posture_consistency()` at `tests/policy/test_release_posture_consistency.py:84-103` calls `splitlines()` and compares every forbidden phrase only with one `lowered` physical line. The committed fixtures preserve each critical phrase on one line, so they do not challenge this boundary.

Auditor-executed temporary fixtures produced:

```text
`v1.0.0-rc15` is the
current public Windows beta.

Use only the public
`v1.0.0-rc15` release.

`v1.0.0-rc16` is
unpublished and not approved.
```

The exact committed evaluator returned `WRAPPED_VIOLATIONS=0`. Each statement remains an operative prior-release directive prohibited by the module contract and round-4 acceptance criterion; only source reflow changed.

**Consequence:** routine editor/formatter wrapping, or a future copy edit, can restore the same public release-truth defect while the regression suite remains green. This is the same false-negative class the guard exists to close.

**Smallest acceptance criterion:** evaluate logical Markdown blocks (or otherwise normalize intra-block whitespace) before phrase matching while retaining a source-line location for findings. Add red fixtures that split each core family at meaningful boundaries, including between “is”/“unpublished,” “the”/“current,” and “public”/the version tag. Prove those fixtures and the exact round-4 parent red, legal historical statements green, the corrected repository green, and rerun the doc/policy blast radius.
