## Findings

### CC-RC17-005 — Major: active release surfaces still direct users to rc15 or describe rc16 as unpublished

**Dimensions:** Documentation, UX, Tests

**Evidence:**

- `ARCHITECTURE.md:3-7` declares rc17 current with published signed assets, while `ARCHITECTURE.md:12-15` calls rc15 the current public repair beta.
- `README.md:20-22` and `README.md:38-40` declare rc17 current, while `README.md:7`, `README.md:135-138`, and `README.md:142` identify rc15 as the public beta, say to use only rc15, publish rc15's exact byte size and SHA-256, and direct every new installer rehearsal to rc15.
- `FAQ.md:18-20` says rc15 is the current public Windows repair beta despite the rc17 capability posture later in the same active FAQ.
- `SUPPORT.md:3-6` declares rc17 current, while `SUPPORT.md:32-35` says rc15 is current and exact-public validated.
- `docs/tester/SMARTSCREEN-WALKTHROUGH.md:3-4` declares rc17 current, while lines 6-7 say rc16 remains unpublished and unapproved for that walkthrough.
- `docs/tester/known-limitations.md:3-23` says rc15 is the validated/approved repair release and rc16 is unpublished, while lines 134-136 say rc17 is current.
- Despite those exact contradictions, `scripts/policy/check_release_identity.py`, `scripts/policy/check_release_candidate_boundary.py`, and `tests/policy/test_current_release_candidate_docs.py` all pass. The current-release-doc test derives rc17 but only requires its presence in selected places; its stale-marker scan does not reject prior rc15/rc16 current-release directives. The candidate-boundary control deliberately returns early once current and introduced tags are equal, so it is not a substitute for this missing consistency guard.

**Consequence:** a reader following the repo's front door can download or rehearse the wrong release and reuse rc15-specific identity material while other sections assert rc17. That defeats release-matched acquisition and makes the public posture internally untrustworthy at publication time.

**Smallest acceptance criterion:** update every active current-release instruction to rc17 or explicitly label it as historical rc15/rc16 evidence; remove rc15-specific acquisition identity from the current **Start here** path; and add a regression control that fails when any curated active release surface calls rc15 current/approved/only-use or calls rc16 unpublished after the current version is rc17. The control must fail against this exact SHA's document state and pass on the corrected descendant.
