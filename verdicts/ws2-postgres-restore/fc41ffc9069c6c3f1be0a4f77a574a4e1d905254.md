## Findings

### CC-WS2-001 - Critical - The equivalence report still has semantic and role/permission false-pass paths

- **Dimension:** correctness / data integrity / required verification
- **Evidence, DDL:** `civiccast/dr/restore_drill.py:305-332` strips every parenthesis and every cast matched by `_CAST_PATTERN` before comparing constraints and indexes. The auditor ran the production `_def_lists_mismatch` at the audited SHA. It returned `None` for all three pairs below:
  - `CHECK ((((a > 0) OR (b > 0)) AND c))` versus `CHECK (((a > 0) OR ((b > 0) AND c)))`; parentheses determine different Boolean semantics.
  - `CHECK ((x)::text IS NULL)` versus `CHECK ((x)::text IS NOT NULL)`; the cast regex greedily consumes the following `IS [NOT] NULL`, normalizing both to `CHECKx`.
  - an index on `amount::integer` versus the same named index on `amount::numeric`; both normalize to the same string although their key values/types differ.
- **Evidence, roles and permissions:** `civiccast/dr/restore_drill.py:252-302` observes only table grants and source table owners. It does not compare database, schema, sequence, or restored-object ownership/privileges, role attributes, or role memberships. `run_postgres_restore` still uses `--no-owner`. `run_postgres_restore_drill` restores a second database in the same cluster, so every source role already exists. Lines 538-562 read `globals.sql` but never replay it. Lines 374-396 only regex-search for `CREATE ROLE`/`ALTER ROLE`; the production helper returned `True` for `ALTER ROLE station` (truncated), `ALTER ROLE station WITH NOLOGIN NOSUPERUSER` (potentially wrong attributes), and `-- CREATE ROLE station;` (comment only).
- **Consequence:** a drill can report `ok=True` after a constraint changes meaning, an index changes its key semantics, or a copied backup cannot recreate the station's roles/ownership/privileges on a fresh cluster. This directly affects post-restore writes, authorization, DR, cold standby, and later cutover.
- **Smallest acceptance criterion:** replace destructive text stripping with semantic canonicalization that preserves Boolean grouping, cast types, null predicates, collation/order, operators, literals, columns, and targets. The coder's recorded same-server canonicalization alternative is acceptable; an AST-aware equivalent is also acceptable. Add exact expected-red controls for the three collisions above. Separately, replay a validated `globals.sql` into an independently fresh/roleless cluster before restoring the database artifact; compare relevant role attributes and memberships plus database/schema/table/sequence owners and privileges on source versus destination. Mutated or truncated globals, wrong role attributes/membership, and revoked object grants must fail the real report. Mere regex presence is not proof of restorability.
- **Blast radius:** disaster recovery, cold standby, database authorization, referential/check integrity, uniqueness/query plans, and native cutover.

### CC-WS2-003 - Major - Two endpoint snapshots do not bind the manifest to `pg_dump`'s snapshot

- **Dimension:** correctness / proof soundness
- **Evidence:** `civiccast/dr/backup.py:498-519` performs `snapshot_tables`, then a separate `pg_dump`, then another `snapshot_tables`; it does not share an exported MVCC snapshot, acquire a database/application maintenance lock, or block writers. `_assert_backup_quiescent` compares only the two endpoint table snapshots. The auditor drove the real `run_full_backup` orchestration with identical pre/post snapshots while the mocked dump produced arbitrary intermediate-state bytes; the function accepted the manifest (`MANIFEST_ACCEPTED=True`). The same logical hole exists with a real ABA write: change data after the first snapshot, let `pg_dump` observe it, then restore the original data before the second snapshot.
- **Consequence:** `before == after` can coexist with dump bytes representing neither endpoint. The manifest can therefore describe one state while the artifact restores another, yet the advertised quiescence proof passes.
- **Smallest acceptance criterion:** bind `snapshot_tables` and `pg_dump` to one exported PostgreSQL snapshot, or acquire and hold an enforceable maintenance/write interlock that every writer honors from before the first snapshot through dump completion and manifest capture. Add a real-Postgres concurrent-write/ABA negative control against `run_full_backup`; the current endpoint-only implementation must go red and the chosen binding must make the artifact and manifest represent one state.
- **Blast radius:** restore comparisons, backup trust, cold-standby initialization, and any operator decision based on `manifest.json`.
