## Executive result

The round-5 implementation changes close both functional false-pass paths identified at `ac6336a4`. Membership comparison now retains every edge with either endpoint in the source-relevant role set, so an unexpected standby-local role connected to a relevant role appears on exactly one side of the symmetric comparison. The table ACL collector now uses `aclexplode` with `acldefault` and compares grantee (including PUBLIC), grantor, table, privilege, and grantability. Direct auditor controls against the production helpers confirmed both corrections. An edge touching no relevant role is a valid scope boundary: any chain that changes authorization for a relevant role necessarily has a first edge touching that role or its source closure, and that edge is retained.

The gate nevertheless remains incomplete because the exact false-pass fixes have no committed report-level negative controls. The prior round's acceptance criterion explicitly required a real `run_postgres_cold_standby_drill` expected-red control for a standby-only membership edge and controls for a PUBLIC table grant and a `WITH GRANT OPTION` difference. This SHA changes only the implementation file; the test inventory and CI floor remain 17. CI therefore proves all earlier controls execute, but it never drives these new branches through the real report. For a charter gate whose deliverable is commit-bound equivalence proof, that is a functional verification gap, not a wording issue.

Under the owner's severity calibration, the missing executable gate proof is Major and blocks this round. The implementation corrections themselves are accepted and do not need redesign.
