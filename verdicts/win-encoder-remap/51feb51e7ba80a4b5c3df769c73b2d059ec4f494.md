### 3. Start and reload share pre-flight - proved

Both `GstPlayoutStrategy.start()` and `reload_content()` call
`_resolve_encoder_override()` and pass the result to `graph_from_config()`.
The positive reload test passed. Replacing the reload resolver result with
`None` in a disposable exact-SHA mutation worktree made
`test_preflight_reload_content_preserves_software_fallback` fail because the
reload graph no longer contained `openh264enc`.

### 4. H.264 fallback and refusal - proved

With hardware absent and fallback enabled, the serialized graph contains
`openh264enc` and excludes `mfh264enc`. With fallback disabled,
`EncoderUnavailableError` is raised. Its operator-facing message names the
channel setting, CPU behavior, and restart remediation.

### 5. Native-Windows HEVC refusal and software x265 exclusion - proved

The software-encoder set is checked before the hardware `"265"` refusal.
`libx265` resolves to `x265enc`, bypasses the hardware probe, and proceeds
unchanged. Hardware HEVC is refused. Moving the `"265"` check before the
software set made the named x265 regression test fail as expected.

### 6. Media Foundation H.264 runtime - proved at confidence class 3

Using
`C:\dev\civiccast-session0-spike\venv\Scripts\python.exe` with
`gstreamer-bundle==1.28.5`, the auditor reran the committed proof:

- `mfh264enc` klass: `Codec/Encoder/Video/Hardware`
- encode: EOS, 5,947,756 bytes, approximately 9.52 Mbps
- decode-back through `tsdemux ! h264parse ! avdec_h264`: EOS
- output SHA-256 prefix: `05af0e8eac270227`

The rerun matched the committed evidence. It also reproduced the HEVC boundary:
unconstrained `mfh265enc` failed with an internal data-stream error, while an
NV12-pinned chain reached EOS.

### 7. Migration reversibility and old payload compatibility - proved

The focused panel drove a real Alembic upgrade-to-head and downgrade-to-0071
against SQLite. Revision 0072 added and then removed
`allow_software_fallback`. `EgressConfig` retains `extra="forbid"` and the new
field defaults to `False`; old payloads omitting it validated successfully.
The exact-SHA Ubuntu `ci-test` job passed its PostgreSQL-backed ordered suite.
