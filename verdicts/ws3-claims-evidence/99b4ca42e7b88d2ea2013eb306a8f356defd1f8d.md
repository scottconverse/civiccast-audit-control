## Findings

### CC-WS3-008 - Major - Strict locality is bypassable through normal Markdown list forms

- **Dimension:** claim-to-evidence association / Markdown structure / functional false pass
- **Code evidence:** `scripts/policy/check_claims_evidence.py:696-713` splits only lines matching `^\s{0,4}[*-]\s`. `scan_marker_violations` at lines 789-810 relies on those runs as the capability-local sub-block boundary.
- **Executed controls:** the committed `*` sibling case and an independent `-` equivalent report uncovered `restore`; a strong-claim item carrying its own marker returns no violations.
- **Executed falsifications:** replacing the list syntax with `+`, `1.`/`2.`, `1)`/`2)`, blockquoted `> *`, or a valid nested `*` item indented five spaces caused the unmarked claim item and its sibling marker to remain one sub-block. Every variant returned `[]` even though the claim item had no marker of its own.
- **Real-surface exposure:** `docs/releases/0.5.0-dr-drill-verification.md:130` uses `+` list syntax, and `.agent-runs/native-windows/spike-decision-gate/evidence/native-engine-gate.md:11-22` uses ordered list syntax. Both are enforced governed Markdown. Their current occurrences are not false passes; they demonstrate that these are live authoring forms, not invented syntax outside the governed set.
- **Consequence:** a future strong capability claim written in an already-used Markdown list style can be satisfied by a marker on a different list item, leaving `claims-evidence: PASS` after the relevant association is absent.
- **Smallest acceptance criterion:** add expected-red sibling-marker regressions for at least `+`, ordered-dot, ordered-paren, blockquoted, and valid deeper-nested Markdown items, with own-marker positive companions. Then either parse all Markdown list forms allowed on governed surfaces with true item locality, or fail closed when a strong capability claim/marker occurs in a list form the locality parser does not support. Preserve the real-registry, real-marker-removal, and Python-bullet regressions.
- **Blast radius:** every enforced Markdown governed document and future additions; shared `_bullet_split` behavior for Python governed docstrings; D1 association for generic capability words.
