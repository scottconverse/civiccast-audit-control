- **Surface:** `.agent-runs/native-windows/ws1-release-truth/evidence/run_evidence.py:6-9`, `:48-52`, `:71-74`; `build-verification.log:3-7`
- **Evidence:** `git_blob_hash` executes `git hash-object <file>`. That form hashes the working bytes directly; it does not supply the path required to apply the repository's clean filter. A CRLF mutation of the committed checker blob produced:

  - `git hash-object --stdin --path=scripts/policy/check_release_truth.py` over the same CRLF bytes: `9c9cbb7ed334e394ff676eb833058c98e01923b8` (match).
- **Consequence:** the current four log lines match because this repository's checked-out files are LF, but the runner and committed evidence text claim checkout-independent CRLF/LF identity that the exact implemented command does not provide. This is the same normalization contract the round-2 finding required and the PR body now represents as closed.
- **Acceptance criterion:** use the path-aware form, for example `git hash-object --path <rel-path> <rel-path>`, then commit a regression/negative control that feeds or checks asserted CRLF bytes for an LF-committed file, requires the filter-aware hash to equal the committed blob, and requires substantive tampering to differ.
