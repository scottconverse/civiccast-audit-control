## Findings

### CC-WS5-007 — Critical — module-level class resolves, but the default SCM host still cannot build production dependencies

`civiccast/native/supervisor/service_host.py:69` creates
`_service_factory = build_production_service_factory()` at module import, and
`SvcDoRun` calls it at `service_host.py:120`. That correctly moves factory
creation into the SCM host process and the class itself is now import-resolvable.

The production call still reaches `service.default_dependency_provider()` at
`civiccast/native/supervisor/service.py:964-981`, which unconditionally raises
`NotImplementedError`. `build_production_service_factory()` at
`service.py:984-1006` cannot build a `SupervisorService` until that provider
returns the real guard, alert outbox, Postgres/NATS/control-plane probes, and
ProgramData root.

The auditor's direct default-path falsification returned:

```text
factory_module= civiccast.native.supervisor.service
NotImplementedError: production supervisor dependencies (guard monitor,
alerting Session, DB/socket/HTTP probes) are the owner-VM integration seam ...
```

`tests/native/test_supervisor_service_win.py:257-297` passes only after replacing
`service_host._service_factory` with a fake. The test at lines 299-330 similarly
replaces it with a raising fake and asserts the exception. Neither test executes
the committed default provider. Consequently, actual `SvcDoRun` cannot start the
supervision loop under SCM, LocalSystem, an interactive console, or any other
host.

This is the same unclosed half named in the round-3 verdict, whose required
acceptance was a module-level class whose `SvcDoRun` constructs the **concrete**
production dependencies in the host process. Moving the closure fixes process
ownership but not the missing composition.

**Required acceptance:** implement the concrete default dependency provider (or
an equivalently committed default host composition) so the unpatched
module-level `SvcDoRun` builds a real `SupervisorService`; add a test that drives
the actual default composition far enough to prove it no longer raises; then run
the separately owner-VM-bound SCM registration, LocalSystem/session-0, and live
child/control-pipe proof.

**Blast radius:** the entire supervisor runtime. The SCM class can now be loaded,
but supervision, guard enforcement, child containment/recovery, the control
pipe, maintenance, and admin commands never start on the real default path.
