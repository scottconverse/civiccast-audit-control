## Findings

### CC-EC-001 - Major - enabled HEVC cannot start with caption embedding

**Files:** `civiccast/egress/gst/encoder_probe.py:122-125`,
`civiccast/egress/gst/strategy.py:462-469,560-562`,
`civiccast/egress/gst/graph.py:78-102`,
`civiccast/egress/gst/engine.py:397-408`

`decide_encoder()` now permits present `mfh265enc`, but
`CIVICCAST_EGRESS_EMBED_CAPTIONS=1` still attaches the codec-independent caption
leg containing `h264ccinserter` and `h264parse`. Real GStreamer 1.28.5 pad/link
inspection established that the H.265 path cannot link from `cccombiner` to
`h264ccinserter`; engine construction raises on that failed link. The committed
runtime proof and new strategy test both omit captions.

**Consequence / blast radius:** native-Windows HEVC start and reload fail
deterministically whenever the supported caption-embedding mode is enabled.
H.264 is unaffected.

**Acceptance criterion:** add a codec-aware H.265 caption leg with real
encode/embed/decode proof, or refuse HEVC before worker construction whenever
caption embedding is enabled. Cover start and reload for this combination.

### CC-EC-002 - Major - checkbox copy falsely promises HEVC software fallback

**File:** `civiccast/apps/portal-operator/src/screens/ChannelOpsScreen.tsx:792-795`

The help text says that when no hardware video encoder is present CivicCast
will encode on the CPU. `encoder_probe.py:126-131` correctly refuses absent
HEVC hardware even when the checkbox is selected because no software HEVC
ships. An HEVC operator is therefore promised recovery behavior the product
cannot provide.

**Acceptance criterion:** make the copy codec-accurate, for example:
“For H.264 channels, use CPU encoding when no hardware encoder is available
(slower). HEVC/H.265 still requires hardware.”

### CC-EC-003 - Minor - new operator control lacks frontend regression coverage

The existing channel-config Playwright flow does not toggle or assert
`allow_software_fallback`, and operator build/accessibility workflows do not
trigger for PRs targeting `program/native-windows`. The auditor independently
ran `npm ci` and the production TypeScript/Vite build successfully, but no
rendered browser or accessibility flow exercised this control.

### CC-EC-004 - Minor - committed runtime evidence is not self-verifying

`encoder_fixups_proof.py` does not bind or print HEAD, does not assert its
expected results, can print `FAIL` and still exit zero, reuses fixed temporary
outputs, and has a changed-file Ruff F401 error. Its log omits SHA, command,
timestamp, exit status, and hashes. The auditor's direct run reproduced the
claimed observations, so this is evidence-quality debt rather than the basis
for CC-EC-001.
